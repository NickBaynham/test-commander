# `/tc:learn-from-tests`

The Phase 3 test-coverage-ingestion command. Walks pytest-style Python and Playwright spec files under the configured tests root, captures test functions plus the symbols each pytest body references, routes gap signals (`unsupported-test-runner`, `untested-function`) to `<workspace>/requirements/open-questions.md`, contributes a `## From tests` section to `entities.md` (covered classes from `code-derived-model.md` with coverage-confidence annotation), and regenerates `system-model.md` via the shared synthesizer. This is the final `/tc:learn-from-*` command in Phase 3 — after this sub-step, all five learn helpers are shipped.

## Inputs

- `<workspace>/documents/uploaded/tests/` (default; override via `tc-knowledge.tests.source-root`) - source root searched recursively for `test_*.py`, `*_test.py`, and `*.spec.ts`.
- `<workspace>/product-knowledge/code-derived-model.md` (optional) - if generated by `/tc:learn-from-code`, the helper imports `extract_knowledge_from_code` and cross-checks the covered-symbols aggregate against parsed functions; unreferenced public functions emit `untested-function` gaps.
- `<workspace>/config.yaml` (optional) - `tc-knowledge.tests:` configures `source-root` and `ignored-paths`.

## Outputs

| Path | Mode | Owner |
| --- | --- | --- |
| `<workspace>/product-knowledge/tests-coverage.md` | overwrite | this command |
| `<workspace>/product-knowledge/entities.md` (`## From tests`) | section-overwrite | this command |
| `<workspace>/product-knowledge/system-model.md` | overwrite (synthesizer) | shared |
| `<workspace>/requirements/open-questions.md` | append, dedup by `(source-id, question)` | this command |

`user-journeys.md`, `assumptions.md`, and `business-rules.md` are NOT touched. Tests encode no journeys, are confirmed facts not inferences, and express coverage rather than rules.

## Preconditions

- `<workspace>/.test-commander/` exists.
- For the `untested-function` gap to fire, `code-derived-model.md` must be generated by `/tc:learn-from-code` first. Without it, the cross-check is silent.

## Behavior

1. Resolve the workspace under `<project-root>/.test-commander/`.
2. Load `tc-knowledge.tests:` extensions; fall back to documented defaults.
3. Resolve the source root: `<workspace>/<source-root>` (default `documents/uploaded/tests`).
4. Walk the source root for pytest files (`test_*.py`, `*_test.py`) and parse each with `ast.parse`. For every `ast.FunctionDef` / `ast.AsyncFunctionDef` whose name starts with `test_`, capture `(name, source, line)` and the union of `ast.Name.id` + `ast.Attribute.attr` references inside the body.
5. Walk the same root for Playwright spec files (`*.spec.ts`). Count `test('...')` and `test("...")` calls via regex; do not parse the TypeScript. Emit one `unsupported-test-runner` gap per file.
6. If `code-derived-model.md` is generated (carries the `Auto-generated by /tc:learn-from-code` marker and is not the empty-run stub), import `extract_knowledge_from_code` and call its `aggregate()` to get parsed functions and classes. For each public function not referenced by any test, emit `untested-function`. Build the covered/uncovered class lists for the cross-cutting render.
7. Render `tests-coverage.md` (overwrite) with source-files table + test-functions table + covered-symbols aggregate + class-coverage breakdown + gap-signals section.
8. Render the `## From tests` section body for `entities.md` (each class from code-derived-model with its coverage-confidence annotation); call `update_cross_cutting()` which preserves every other source's section.
9. Append open questions for each gap signal, deduplicated by `(source-id, question-text)` (the question text includes the gap kind as a prefix).
10. Call `synthesize_system_model.synthesize(project_root)` to regenerate `system-model.md`.
11. Exit 0.

## Safety

- The helper reads only from the configured tests root and the workspace's per-source models. No network calls; no shell-out.
- `ast.parse` is the standard CPython parser; no code execution.
- Re-running against unchanged input is byte-deterministic across every output.
- Per the Phase-3 design decisions, this helper does not write to `<workspace>/traceability/`.
- Per D19, the detection patterns carry no domain vocabulary.

## Implementation

- Helper: `plugins/test-commander/scripts/extract_knowledge_from_tests.py` (~600 lines).
- Reuses `extract_knowledge_from_code.aggregate()` for the cross-check via Python import (mirrors Step 3.5's cross-helper import pattern).
- Tests: `tests/test_learn_from_tests.py` (23 cases - uninitialized refused, no-tests stub, pytest extraction (files + functions + provenance + covered-symbols), Playwright detection + `unsupported-test-runner` always, `untested-function` after code helper runs, cross-check skipped without code model, cross-cutting scope (entities; NOT user-journeys / assumptions / business-rules), prior-section preservation across all four prior helpers, full idempotency, `source-root` extension, synthesizer reflects).

## Definition of Done

- Helper passes all 23 test cases.
- Pytest AST extraction and Playwright counting both work.
- Cross-check against `code-derived-model.md` is conditional and correct.
- Methodology covers all 3 positive dimensions plus both gap signals with worked examples and Claude-judgment-layer paragraphs.
- Template authored.
- Per-command page complete (this file).
- `tests-coverage.md` added to the Workspace Layout block in `planning/plan.md` and to `docs/workspace-reference.md`.
- `tc-knowledge/SKILL.md` describes all five Phase 3 commands plus the shared synthesizer with no deferral wording.
- `make verify` chain green.
