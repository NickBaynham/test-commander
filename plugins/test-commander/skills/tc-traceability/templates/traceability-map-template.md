# Traceability map shapes

`/tc:traceability-map` regenerates two files under `<workspace>/traceability/`.
This template documents both shapes.

## `requirements-map.md` (shared 4-column format)

```
# Requirements Map

Cross-references each parsed requirement to the downstream artifacts that trace
back to it. Generated by Test Commander (`/tc:requirements-coverage` and
`/tc:traceability-map`); overwritten on every run.

| REQ-ID | Test ideas | BDD features | Automation |
| --- | --- | --- | --- |
| REQ-001 | `test-ideas/REQ-001.md` | `bdd/features/<area>.feature` | _(none)_ |
```

Byte-identical whichever helper wrote it (the shared renderer).

## `test-map.md` (scenario-level chain)

```
# Test Map

Scenario-level traceability ... Downstream links render `pending` until Phase 6
(automated tests) and Phase 7 (results, quality report) populate them.

| Requirement | Test idea | BDD scenario | Automated test | Test result | Quality report |
| --- | --- | --- | --- | --- | --- |
| REQ-001 | CS-001-001 | `bdd/features/<area>.feature` :: <scenario name> | pending | pending | pending |
```

Empty case: `_No BDD scenarios with linkage tags found._`
