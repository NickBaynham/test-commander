# Vision

Test Commander exists to make AI-assisted testing safer, more useful, and more traceable than the default behavior of letting a model "just write tests."

Modern test automation tools assume someone has already decided what to test. Test Commander challenges that assumption. It starts from product understanding — requirements, user stories, the application itself — and works forward through exploration, ideas, BDD, automation, evidence, and reporting. Every artifact ties back to the next one, so a quality report is never disconnected from the requirement that motivated it.

The north star: turn requirements, exploration, automation, evidence, reporting, and continuous learning into one visible workflow that a team can trust and steer.

## What we are not building

- A fully autonomous QA agent that ships tests on its own.
- A wrapper over third-party plugins. We own every skill in-repo.
- A test runner. Playwright is the runner; we orchestrate it.
- A replacement for skilled testers. We are their amplifier.

## Tone

Test Commander learns continuously, but improves deliberately. Every change to the methodology passes through human review.
