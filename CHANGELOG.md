## [Unreleased]

### Tooling / docs

- New one-line `CLAUDE.md` at the repo root containing `@AGENTS.md`. Claude Code's project-instruction loader walks up the directory tree looking for `CLAUDE.md` by name; without this file the per-project `AGENTS.md` was discoverable (a human or directed agent could find it via the README) but **not auto-loaded** into the session's system context. The one-line file uses Claude Code's `@<path>` import directive so `CLAUDE.md` loads `AGENTS.md` as project instructions, keeping `AGENTS.md` as the single canonical (vendor-neutral) source of truth and `CLAUDE.md` as the auto-load shim. Pattern matches `testingwithagents/Claude.md` which uses the same delegate. Verified: link checker covers 109 files (up from 108 — `CLAUDE.md` added with its `@` directive); the directive is not a Markdown link so it does not introduce link-check noise.

- New `AGENTS.md` at the repo root — the entry-point document an agent (Claude or any automated operator) reads at the start of every session before touching code. Without it, every new session had to re-discover the project's disciplines by reading the 1,900-line `planning/plan.md` end-to-end, reverse-engineering commit patterns from `git log`, and re-learning the TDD micro-cycle / verify chain / sign-off pattern. AGENTS.md captures: project identity in one paragraph (Test Commander as a domain-agnostic AI-assisted testing system); source-of-truth table pointing at the plan, CHANGELOG, README, docs, and the plugin skill directory; quick-reference one-liners for Decisions D1–D19; the seven Per-Phase Conventions with the full rule text (six per-phase deliverables, `claude` CLI for plugin ops, retiring prior-phase guards, SKILL.md surfaces shipped behavior, per-command page as single source of truth, customization-guide audit per D19, sub-step lesson capture); the strict 11-step TDD micro-cycle (write tests → red → implement → green → command page → methodology/template → SKILL.md update → make verify → lesson capture → CHANGELOG → commit); the helper-mirroring pattern that produced Step 2.3's 9/9-on-first-run result; the verify chain (ruff, pytest, verify_skills.py, check_links.py); commit conventions (HEREDOC multi-paragraph, `Co-Authored-By: Claude Opus 4.7` line per this repo's convention, per-sub-step commits, `plan:` prefix for plan-only commits); the six sub-sub-step phase sign-off pattern (cold-user walkthrough, per-step DoD audit, plan + CHANGELOG closing, doc final pass, test-first sign-off test with `>=` invariants, final DoD eval + annotated tag); the per-sub-step Definition of Done; a "What NOT to do" section catching the project's known anti-patterns (no `git add -A`, no exact-equals in sign-off tests, no template-stub-vs-generated confusion, no skip-not-overwrite on pure reports, no domain-specific vocabulary in shipped defaults, no deferral wording in shipped SKILL.md, no emojis); and a "When this orientation is wrong" closing — if drift is hit, the runbook is updated in the same PR. Cross-linked from `README.md` "For agents" section so human readers also discover it. The motivation is the same as the testingwithagents `docs/AGENT_BLOG_PUBLISHING.md` runbook from 2026-05-27: capture the operational knowledge once so the next session does not pay the rediscovery cost.
